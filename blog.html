Important FAQ

What is the Document Object Model (DOM)?

DOM means Document Object Model. It is used as a bridge to connect html and javascript. Using DOM, we can utilize javascript functions to target an element inside html. DOM converts the html file into a tree structure. Each element of the elements are called nodes in terms of DOM. Using javascript we can target those nodes and dynamically manipulate them. We can change the way they behave, increase or decrease their height/width or any other styles and css. We can add classes, id, insert new elements or even delete nodes from the html file.

How do you select an element from the DOM?

In the DOM, we can select elements using several methods provided by JavaScript. Here are the most common ways to select elements:
1. document.getElementById() Selects an element by its id attribute. This method returns a single element.
2. document.getElementsByClassName() It returns an HTMLCollection (an array-like object), and we can access individual elements using indices.
3. document.getElementsByTagName() Selects all elements with a given tag name. Like getElementsByClassName(), it returns an HTMLCollection.
4. document.querySelector() It is very useful to specifically target an element using css selectors. We don't need any id, or class to do this task.
5. document.querySelectorAll() Selects all elements that match a specified CSS selector and returns a NodeList. We can use forEach method inside a nodelist.

What is event delegation in the context of the DOM, and why is it useful?

Event delegation is a technique where the parent element does the work which its children are supposed to do. For example, if a container has multiple buttons, then instead of attaching eventListener to all of the buttons, we can add only a single listener for the parent, we can use event parameter to look at the target element and still do the work intended for the buttons.

    Why Event Delegation Is Useful:
  • 1. Improved Performance: If we have many child elements (e.g., hundreds of list items or buttons), attaching individual event listeners to each one can be inefficient and memory-consuming. Event delegation attaches only one event listener to the parent, improving performance.
  • 2. Handles Dynamically Added Elements: With event delegation, we donâ€™t need to reattach event listeners for new elements added to the DOM. Parent element will do the work even if we dynamically add new children inside the parent.
  • 3. Cleaner Code: Even delegation helps us maintain and organize our code in a better way.
How do you manipulate an element's attributes and styles using the DOM?

  • 1. Manipulating Attributes:We can use getAttribute() method to get the attributes and to set or update an attribute, we can use the setAttribute() method.
  • 2. Manipulating Styles: We can directly manipulate an element's style by modifying the style property. For example, document.getElementById("container").style.color ="white" will set the text color of the div with id container to white.
  • 3. Computed Styles: If we want to get the current style of an element, we can use getComputedStyle().
  • 4. Style Object: We can target an element and use style object and its properties to dynamically change an element's design using JS.
