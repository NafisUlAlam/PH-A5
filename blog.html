Important FAQ

What is the Document Object Model (DOM)?

DOM means Document Object Model. It is used as a bridge to connect html and javascript. Using DOM, we can utilize javascript functions to target an element inside html.

How do you select an element from the DOM?

In the DOM, we can select elements using several methods provided by JavaScript. Here are the most common ways to select elements:
1. document.getElementById() Selects an element by its id attribute. This method returns a single element because id is supposed to be unique within a document.
2. document.getElementsByClassName() Selects all elements with a specific class. It returns an HTMLCollection (an array-like object), and we can access individual elements using indices.
3. document.getElementsByTagName() Selects all elements with a given tag name. Like getElementsByClassName(), it returns an HTMLCollection.
4. document.querySelector() Selects the first element that matches a specified CSS selector. It’s very flexible because we can use any valid CSS selector (like class, id, or tag).
5. document.querySelectorAll() Selects all elements that match a specified CSS selector and returns a NodeList (a static collection).

What is event delegation in the context of the DOM, and why is it useful?

Event delegation is a technique in the DOM that allows us to handle events more efficiently by attaching a single event listener to a parent element instead of adding listeners to individual child elements. This parent element listens for events on its children by taking advantage of event bubbling. We can move within the parent and its children using event target and parentElement.

    Why Event Delegation Is Useful:
  • 1. Improved Performance: If we have many child elements (e.g., hundreds of list items or buttons), attaching individual event listeners to each one can be inefficient and memory-consuming. Event delegation attaches only one event listener to the parent, improving performance.
  • 2. Handles Dynamically Added Elements: With event delegation, we don’t need to reattach event listeners for new elements added to the DOM. Since the parent element handles the event, it automatically works for dynamically created child elements as well.
  • 3. Cleaner Code: Instead of attaching and managing multiple listeners, event delegation centralizes event handling in one place, making the code easier to manage and maintain.
How do you manipulate an element's attributes and styles using the DOM?

  • 1. Manipulating Attributes: To retrieve the value of an attribute from an element, we can use the getAttribute() method. To set or update an attribute, we can use the setAttribute() method.
  • 2. Manipulating Styles: We can directly manipulate an element's style by modifying the style property. For example, document.getElementById("container").style.color ="white" will set the text color of the div with id container to white.
  • 3. Accessing Computed Styles: If we want to get the current style of an element (including styles set via external CSS), we can use getComputedStyle().
